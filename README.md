# دفترچهٔ رشد — راهنمای کامل

این پروژه یک اپلیکیشن پایتون (Kivy) برای توسعهٔ فردی است که شامل پروفایل،
چک‌لیست روزانه، اهداف ماهانه (بر اساس روودمپ یک‌ساله)، تایمر، یادآوری، و
روتین پوستی است. هدفش پشتیبانی از مسیر یک‌سالهٔ تو در پایتون، آلمانی، ترید،
مکانیک، هک و امنیت، ورزش، و کاهش وزن (۱۱۰ → ۸۵ کیلوگرم) است.

⚠️ **نکتهٔ مهم**: من (Claude) نمی‌توانم خودم یک فایل APK بسازم، چون ساخت APK
نیاز به Android SDK / NDK دارد که دانلودش به اینترنت نیاز دارد و محیط من به
اینترنت دسترسی ندارد. اما کد کامل و آماده را به تو می‌دهم؛ مراحل زیر را دنبال
کن تا خودت (روی کامپیوتر یا با سرویس آنلاین) APK را بسازی و روی تبلتت نصب کنی.

---

## فایل‌های پروژه

```
growthapp/
├── main.py            # کد اصلی اپلیکیشن (Kivy)
├── roadmap.py         # روودمپ یک‌ساله و برنامهٔ روزانه/ماهانه (قابل ویرایش)
├── buildozer.spec     # تنظیمات ساخت APK
├── requirements.txt   # برای تست روی کامپیوتر (دسکتاپ)
├── data/              # محل ذخیرهٔ اطلاعات کاربر (خودکار ساخته می‌شود)
└── fonts/             # محل قرار دادن فونت فارسی (توضیح در پایین)
```

---

## مرحلهٔ ۱: تست روی کامپیوتر (پیشنهاد می‌شود قبل از ساخت APK)

قبل از اینکه برای اندروید بسازی، بهتره مطمئن شی روی کامپیوتر خودت درست اجرا
می‌شه (سریع‌تر و راحت‌تر برای رفع‌اشکال است).

1. پایتون ۳.۹ تا ۳.۱۲ نصب باشد.
2. در پوشهٔ پروژه:
   ```bash
   pip install -r requirements.txt
   python main.py
   ```
3. اگر پنجره باز شد و متن فارسی به‌صورت بهم‌ریخته/جعبه‌ای دیدی (نه خرابی در
   اجرا)، به بخش «فونت فارسی» در پایین برو.

---

## مرحلهٔ ۲: فونت فارسی (مهم!)

Kivy به‌صورت پیش‌فرض فونتی ندارد که حروف فارسی را به‌درستی نشان دهد (چسبیدن
حروف به هم). برای حل این مشکل:

1. یک فونت فارسی رایگان دانلود کن، پیشنهاد: **Vazirmatn**
   (از سایت [fonts.google.com](https://fonts.google.com/specimen/Vazirmatn) یا
   مخزن [rastikerdar/vazirmatn](https://github.com/rastikerdar/vazirmatn) در گیت‌هاب).
2. فایل `Vazirmatn-Regular.ttf` را در پوشهٔ `fonts/` کنار `main.py` قرار بده.
3. کد به‌صورت خودکار این فونت را شناسایی و استفاده می‌کند (در ابتدای `main.py`).

اگر این فایل را نگذاری، برنامه باز هم اجرا می‌شود اما حروف فارسی ممکن است
درست به هم نچسبند (چون Kivy از HarfBuzz/shaping کامل فارسی پشتیبانی محدودی
دارد). در عمل با فونت Vazirmatn نتیجه قابل قبول می‌شود، اما اگر باز هم
شکل حروف ایراد داشت، گزینهٔ جایگزین استفاده از کتابخانه `arabic_reshaper` و
`python-bidi` است (نصب با pip و reshape کردن متن‌ها قبل از نمایش) — اگر این
مرحله لازم شد بگو تا کدش رو هم اضافه کنم.

## مرحلهٔ ۳: ساخت APK با Buildozer

Buildozer فقط روی **لینوکس** (یا WSL در ویندوز، یا macOS) کار می‌کند — روی
ویندوز مستقیم کار نمی‌کند.

### اگر لینوکس داری (یا WSL نصب کردی):

```bash
# نصب پیش‌نیازها (Ubuntu/Debian)
sudo apt update
sudo apt install -y python3-pip build-essential git python3-dev \
    ffmpeg libsdl2-dev libsdl2-image-dev libsdl2-mixer-dev libsdl2-ttf-dev \
    libportmidi-dev libswscale-dev libavformat-dev libavcodec-dev \
    zlib1g-dev openjdk-17-jdk unzip

pip install --upgrade buildozer cython==0.29.36

cd growthapp
buildozer android debug
```

اولین بار که اجرا می‌کنی، Buildozer خودش Android SDK و NDK را دانلود می‌کند
(چندین گیگابایت، ممکن است ۳۰-۶۰ دقیقه طول بکشد بسته به اینترنت). در پایان،
فایل APK در مسیر `bin/growthjournal-1.0-debug.apk` ساخته می‌شود.

### اگر کامپیوتر لینوکس/WSL نداری: سرویس آنلاین (GitHub Actions)

اگر فقط ویندوز داری و نمی‌خوای WSL نصب کنی، می‌تونی از GitHub Actions به‌صورت
رایگان APK بسازی:

1. یک ریپازیتوری گیت‌هاب بساز و فایل‌های پروژه (`main.py`, `roadmap.py`,
   `buildozer.spec`, پوشهٔ `fonts/`) را در آن آپلود کن.
2. یک فایل در مسیر `.github/workflows/build.yml` با این محتوا بساز:

```yaml
name: Build APK
on: [push, workflow_dispatch]
jobs:
  build:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - name: Install system deps
        run: |
          sudo apt update
          sudo apt install -y build-essential git python3-dev ffmpeg \
            libsdl2-dev libsdl2-image-dev libsdl2-mixer-dev libsdl2-ttf-dev \
            libportmidi-dev libswscale-dev libavformat-dev libavcodec-dev \
            zlib1g-dev openjdk-17-jdk unzip
      - name: Install buildozer
        run: pip install --upgrade buildozer cython==0.29.36
      - name: Build APK
        run: buildozer android debug
      - uses: actions/upload-artifact@v4
        with:
          name: growthjournal-apk
          path: bin/*.apk
```

3. این فایل را هم push کن. GitHub Actions به‌صورت خودکار اجرا می‌شود (در تب
   "Actions" ریپازیتوری) و در پایان یک فایل APK قابل دانلود می‌سازد (در بخش
   "Artifacts" همان اجرا).
4. APK را دانلود کن، روی تبلت اندرویدی منتقلش کن (با کابل، ایمیل، یا گوگل
   درایو) و نصبش کن (باید "نصب از منابع ناشناس" را در تبلت فعال کنی).

---

## مرحلهٔ ۴: نصب روی تبلت

1. فایل `.apk` ساخته‌شده را به تبلت منتقل کن.
2. روی فایل APK در تبلت ضربه بزن.
3. اندروید معمولاً هشدار می‌دهد چون اپ از "Google Play" نیست — باید گزینهٔ
   "نصب از این منبع" یا "Install anyway" را تأیید کنی (در تنظیمات امنیتی
   تبلت گزینهٔ "Unknown sources" / "نصب اپ‌های ناشناس" را برای همان مرورگر
   یا فایل‌منیجر فعال کن).
4. بعد از نصب، آیکون "Growth Journal" در صفحهٔ اصلی ظاهر می‌شود.

## دربارهٔ نوتیفیکیشن (نکتهٔ مهم محدودیت)

کد فعلی با کتابخانهٔ `plyer` کار می‌کند که:
- ✅ **وقتی اپ باز است**: هر ۳ ساعت یک یادآوری محلی می‌فرستد (قابل تغییر در
  `main.py`، تابع `_periodic_reminder` و عدد `60*60*3`).
- ⚠️ **وقتی اپ کاملاً بسته است**: نوتیف ارسال نمی‌شود. برای این حالت، اندروید
  نیاز به یک **سرویس پس‌زمینه** (Foreground Service) دارد که پیچیده‌تر است و
  باید جدا تنظیم شود (نیاز به `pyjnius` و کدنویسی مستقیم با Android API). اگر
  این قابلیت برات حیاتیه، بگو تا در مرحلهٔ بعد به پروژه اضافهش کنم — فقط بدان
  که این بخش از قسمت‌های شکننده‌تر Kivy روی اندروید است و معمولاً نیاز به چند
  دور تست-و-خطا روی گوشی واقعی دارد.

## ویرایش روودمپ یا برنامهٔ روزانه

تمام برنامهٔ روزانه/ماهانه در فایل `roadmap.py` است — می‌توانی متن تسک‌ها،
محور هر روز هفته، یا اهداف هر ماه را مستقیماً در آن فایل ویرایش کنی، بدون
نیاز به تغییر `main.py`.

## مشکلات رایج

| مشکل | راه‌حل احتمالی |
|---|---|
| حروف فارسی بهم‌ریخته | فونت Vazirmatn را در `fonts/` قرار بده (مرحلهٔ ۲) |
| Buildozer خطای SDK می‌دهد | دوباره اجرا کن؛ گاهی دانلود اول قطع می‌شود |
| اپ روی تبلت crash می‌کند | با `buildozer android debug deploy run logcat` لاگ را روی گوشی متصل‌شده ببین |
| نوتیف نمی‌آد وقتی اپ بسته‌ست | طبیعی است؛ به بخش «دربارهٔ نوتیفیکیشن» مراجعه کن |
